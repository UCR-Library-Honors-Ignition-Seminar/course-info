# GitHub & GitHub Classroom Guide for Students

> **No coding experience required.** Everything in this guide can be done in a web browser.

---

## Part 1: What Is This and Why Do We Use It?

**GitHub** is like Google Drive for your projects, but with automatic version history. Every time you save (called a "commit"), GitHub remembers what your files looked like — so nothing is ever truly lost.

**GitHub Classroom** is the system your instructors use to distribute assignments. When you click the weekly assignment link on Canvas, GitHub Classroom automatically creates a private copy of that week's assignment folder just for you.

**Why does this class use GitHub?**
- It builds a public creative portfolio you can share with employers or graduate schools
- Instructors can see your process, not just your final result
- You will be working with a tool used across industries — design, research, publishing, and tech

## Part 2: Getting Started

### Step 1: Create a GitHub Account

1. Go to [github.com/signup](https://github.com/signup)
2. Use your UCR email address (recommended)
3. Choose a username you are comfortable making public — this will be on your portfolio

### Step 2: Accept the Week's Assignment

Each week, your instructor will post an **assignment invitation link** on Canvas.

1. Click the link on Canvas
2. Sign into GitHub if prompted
3. Click **"Accept this assignment"**
4. GitHub Classroom creates a personal repo for you automatically — no extra setup needed
5. Click the link shown to open your new repo

---

## Part 3: Adding Your Work (Web Browser — No Downloads Needed)

You do not need to install anything. Use GitHub's built-in web editor:

### Step 3: Create a folder and add your reflection

1. Open your assignment repo on GitHub
2. Click **"Add file"** → **"Create new file"**
3. In the filename box, type: `week-05/reflection.md`
   - Typing the `/` automatically creates the folder `week-05`
4. Write or paste your reflection into the text box
5. Click the **"Preview"** tab to see how it will look when rendered
6. When ready, scroll down to **"Commit changes"**
7. Add a short description (e.g., "added Week 5 reflection")
8. Click the green **"Commit changes"** button

### Step 4: Upload project files

1. Navigate into your `week-05` folder (click on it)
2. Click **"Add file"** → **"Upload files"**
3. Drag and drop your file(s) (HTML export, screenshot, PDF, etc.)
4. Add a commit message and click **"Commit changes"**

### Step 5: Submit on Canvas

1. Copy your repo URL from the browser address bar
2. Go to Canvas → that week's assignment → **Submit**
3. Paste the URL and submit

For projects hosted elsewhere (p5.js editor, itch.io, etc.):
- Create a file called `links.md` in your week folder
- Paste the project URL in that file
- Submit the Canvas assignment with your GitHub repo URL

## Part 4: Writing in Markdown

Your reflection goes in a file called `reflection.md`. Markdown is plain text with simple formatting codes.

### Common formatting

```
# Big heading
## Medium heading

**bold text**
*italic text*

- Bullet point
- Another bullet point
```

### Using AI to help format your Markdown

You are **encouraged to use AI tools (ChatGPT, Claude, etc.) to help format your reflection.md file**. For example:

> "Take these notes I wrote and format them as a Markdown file with a heading for each section."

This is not the same as having AI write your reflection — the ideas and critical thinking must be yours. Using AI to clean up formatting is fine and a practical skill.

**How to check your formatting:** Click the **"Preview"** tab in the GitHub editor before committing.

## Part 5: Glossary

| Term | Plain-language definition |
|------|--------------------------|
| **Repository (repo)** | Your project folder on GitHub |
| **Commit** | Saving a version of your file with a note about what changed |
| **Push** | Sending changes to GitHub (automatic when using the web editor) |
| **Fork** | Making your own copy of someone else's repo |
| **README.md** | The description file shown when someone visits your repo |
| **GitHub Pages** | A free website GitHub hosts from your repo's files |
| **Branch** | A parallel version of your repo — you won't need this, just use `main` |

## Part 6: Common Questions and Fixes

**"I can't find my assignment repo."**
Check your GitHub notification email for the Classroom invitation link. Also check Canvas — the link is there each week.

**"I accidentally deleted a file."**
GitHub keeps all previous versions. Your instructors can recover it. Click the "History" button on any file to see past versions.

**"My Markdown doesn't look right."**
Click the **Preview** tab in the editor before committing. Or paste your Markdown into an AI and ask it to fix the formatting.

**"What should I put in the commit message?"**
Just describe what you added: "added week 6 Bitsy game" or "updated reflection" is perfect.

**"Do I need to install Git or any software?"**
No. Everything in this class works in a web browser.

**"My project is a link (p5.js, itch.io), not a file."**
Create a `links.md` file in your week folder and paste the URL there. Then submit your GitHub repo URL on Canvas.
